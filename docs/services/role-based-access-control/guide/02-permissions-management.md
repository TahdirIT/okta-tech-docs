# إدارة الصلاحيات

إدارة الصلاحيات تتيح إنشاء وإدارة الصلاحيات المرجعية بشكل منفصل عن الأدوار. يمكن إنشاء الصلاحيات يدوياً أو استيرادها من الكود تلقائياً.

## الميزات الرئيسية

### 1. إنشاء صلاحية جديدة

يمكن إنشاء صلاحية جديدة يدوياً من خلال:
- **اسم الصلاحية** (`name`): مثل `manage-users`, `view-reports`
- **النطاق** (`scope`): `tenant` أو `system`
- **الحارس** (`guard_name`): افتراضياً `web`

### 2. مسح الكود لاكتشاف الصلاحيات المفقودة

يتم مسح ملفات المشروع لاكتشاف الصلاحيات المستخدمة في الكود ولكن غير موجودة في قاعدة البيانات:

- البحث في ملفات Blade عن `@can`, `@canany`, `@cannot`
- البحث في ملفات PHP عن `can()`, `cannot()`, `hasPermissionTo()`, `hasAnyPermission()`
- البحث عن أنماط مثل `permission:name` في الكود

النتائج تُحفظ في `storage/app/rbac/missing-rbac.json`.

### 3. استيراد الصلاحيات المفقودة

بعد المسح، يمكن:
- **استيراد جميع الصلاحيات المفقودة**: إضافة جميع الصلاحيات المكتشفة إلى قاعدة البيانات
- **استيراد الصلاحيات المحددة**: اختيار صلاحيات محددة من القائمة واستيرادها فقط

### 4. عرض وإدارة الصلاحيات

- عرض جميع الصلاحيات في جدول منظم
- تصنيف الصلاحيات حسب النطاق (`tenant`/`system`)
- حذف الصلاحيات غير المستخدمة

## القواعد والقيود

- يجب أن يكون اسم الصلاحية فريداً ضمن نفس `guard_name` و `scope`
- لا يمكن حذف صلاحية مرتبطة بأدوار أو مستخدمين
- الصلاحيات على مستوى النظام (`system`) متاحة لجميع المستأجرين
- الصلاحيات على مستوى المستأجر (`tenant`) خاصة بكل مستأجر

## استخدام الصلاحيات في الكود

### في Blade Templates

```blade
@can('manage-users')
    <button>إدارة المستخدمين</button>
@endcan

@canany(['view-reports', 'manage-reports'])
    <a href="/reports">التقارير</a>
@endcanany
```

### في Controllers

```php
public function index()
{
    $this->authorize('view-users');
    
    // ...
}
```

### في Livewire Components

```php
public function delete()
{
    $this->authorize('delete-users');
    
    // ...
}
```
