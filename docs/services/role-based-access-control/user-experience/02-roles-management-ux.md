# تجربة المستخدم - إدارة الأدوار

## الوصول إلى الصفحة

يتم الوصول إلى صفحة إدارة الأدوار من القائمة الرئيسية للمنصة تحت قسم "الإعدادات" → "الأدوار"، أو مباشرة عبر المسار `/settings/roles`.

## عرض قائمة الأدوار

عند فتح الصفحة، يرى المستخدم:

### التخطيط العام

الصفحة مقسمة إلى قسمين رئيسيين:

**القسم الأيسر (4 أعمدة):**
- نموذج إضافة دور جديد
- قسم مسح وفحص الأدوار المفقودة

**القسم الأيمن (8 أعمدة):**
- جدول عرض الأدوار

### جدول الأدوار

يعرض الجدول الأعمدة التالية:
- **الاسم (Name)**: اسم الدور (مثل: `superadmin`, `admin`, `teacher`)
- **النطاق (Scope)**: `SYSTEM` أو `TENANT` (يظهر كـ Badge)
- **الصلاحيات (Permissions)**: قائمة بالصلاحيات المرتبطة بالدور (مفصولة بفواصل)
- **الإجراءات (Actions)**: زر حذف الدور

يمكن للمستخدم:
- البحث عن دور معين باستخدام حقل البحث (إن كان متوفراً)
- تصفية النتائج حسب النطاق (System/Tenant)
- عرض الأدوار مرتبة حسب النطاق ثم الاسم

## إنشاء دور جديد

### النموذج

**الحقول:**
- **اسم الدور (Role name)**: حقل نصي لإدخال اسم الدور
  - مثال: `platform-admin`, `content-manager`
  - مطلوب
  - الحد الأقصى: 120 حرف
  
- **النطاق (Scope)**: قائمة منسدلة لاختيار النطاق
  - الخيارات: `Tenant` أو `System`
  - مطلوب
  - القيمة الافتراضية: `Tenant`

- **الصلاحيات المرجعية (Permissions reference)**: قائمة Checkboxes لعرض جميع الصلاحيات المتاحة
  - كل صلاحية تظهر كـ Checkbox مع الاسم والنطاق
  - يمكن اختيار عدة صلاحيات
  - الصلاحيات مرتبة حسب الاسم

### الخطوات

1. المستخدم يملأ حقل اسم الدور
2. يختار النطاق من القائمة المنسدلة
3. يختار الصلاحيات المراد ربطها بالدور من قائمة Checkboxes
4. ينقر على زر "إضافة دور" (Add Role)
5. يتم التحقق من صحة البيانات
6. يتم التحقق من عدم تكرار الاسم في نفس المستأجر
7. إذا نجحت العملية:
   - يتم حفظ الدور في قاعدة البيانات
   - يتم ربط الصلاحيات المحددة بالدور
   - يتم إعادة تعيين النموذج
   - يظهر إشعار نجاح: "تم حفظ الدور بنجاح"
   - يتم تحديث الجدول تلقائياً لعرض الدور الجديد
8. إذا فشلت العملية:
   - يظهر رسالة خطأ توضح السبب
   - يبقى النموذج مفتوحاً مع البيانات المدخلة

### إضافة العناوين (Titles)

**ملاحظة:** حالياً النموذج لا يحتوي على حقول للعناوين، ولكن يجب إضافتها في المستقبل:

- **العنوان بالعربية (title_ar)**: حقل نصي (اختياري)
- **العنوان بالإنجليزية (title_en)**: حقل نصي (اختياري)

## مسح وفحص الأدوار المفقودة

### القسم

يحتوي القسم على:
- عنوان: "مسح واستيراد الأدوار المفقودة"
- وصف: "مسح واستيراد الأدوار المفقودة من الكود إلى قاعدة البيانات"
- زر مسح (أيقونة بحث)
- معلومات آخر مسح (إن كان متوفراً)
- قائمة بالأدوار المفقودة

### الخطوات

1. المستخدم ينقر على زر "مسح" (Scan)
2. يتم فحص الكود للبحث عن الأدوار المستخدمة
3. يتم مقارنة الأدوار المستخدمة مع الأدوار الموجودة في قاعدة البيانات
4. يتم عرض قائمة بالأدوار المفقودة:
   - كل دور يظهر كـ Checkbox مع الاسم
   - يمكن اختيار جميع الأدوار عبر Checkbox "تحديد الكل"
5. المستخدم يختار الأدوار المطلوبة (أو يختار الكل)
6. ينقر على زر "إضافة المحدد إلى قاعدة البيانات" أو "إضافة الكل إلى قاعدة البيانات"
7. يتم استيراد الأدوار المحددة إلى قاعدة البيانات
8. يظهر إشعار نجاح: "تم استيراد الأدوار بنجاح"
9. يتم تحديث الجدول تلقائياً لعرض الأدوار المستوردة

### حالة عدم وجود أدوار مفقودة

إذا لم تكن هناك أدوار مفقودة:
- تظهر رسالة: "لم يتم العثور على أدوار مفقودة. قم بتشغيل المسح للتحديث."

## حذف دور

### الخطوات

1. المستخدم ينقر على زر "حذف" (Delete) بجانب الدور في الجدول
2. يظهر تأكيد (Confirmation Dialog) يسأل عن التأكيد
3. إذا أكد المستخدم:
   - يتم التحقق من عدم استخدام الدور من قبل أي مستخدم
   - إذا كان الدور مستخدماً:
     - يظهر رسالة خطأ: "لا يمكن حذف الدور لأنه مستخدم من قبل مستخدمين"
   - إذا لم يكن الدور مستخدماً:
     - يتم حذف الدور (Soft Delete)
     - يتم إزالة ربط الدور بجميع المستخدمين تلقائياً
     - الصلاحيات المرتبطة بالدور لا يتم حذفها
     - يظهر إشعار نجاح: "تم حذف الدور بنجاح"
     - يتم تحديث الجدول تلقائياً لإزالة الدور المحذوف
4. إذا ألغى المستخدم:
   - لا يتم تنفيذ أي إجراء
   - يبقى الجدول كما هو

### التحذيرات

- يجب التأكد من عدم استخدام الدور من قبل أي مستخدم قبل الحذف
- الأدوار الثابتة (`superadmin`, `admin`, `teacher`, `student`) لا يمكن حذفها
- الحذف يكون Soft Delete (يمكن استرجاعه من قاعدة البيانات)
- عند حذف دور، يتم إزالة ربطه بجميع المستخدمين تلقائياً
- الصلاحيات المرتبطة بالدور لا يتم حذفها

## حالات خاصة

### عدم وجود أدوار

إذا لم تكن هناك أدوار في النظام:
- يظهر في الجدول رسالة: "لا توجد أدوار حتى الآن."
- يظهر زر "إضافة دور" بشكل بارز

### الأدوار على مستوى النظام

- يمكن فقط للمستخدمين الذين لديهم دور `superadmin` إضافة أدوار على مستوى النظام
- الأدوار على مستوى النظام تظهر مع Badge `SYSTEM`
- الدور `superadmin` لا يمكن حذفه

### الأدوار على مستوى المستأجر

- يمكن للمستخدمين الذين لديهم دور `admin` في المستأجر إضافة أدوار على مستوى المستأجر
- الأدوار على مستوى المستأجر تظهر مع Badge `TENANT`
- يتم ربط الدور تلقائياً بالمستأجر النشط في السياق
- الأدوار الثابتة (`admin`, `teacher`, `student`) لا يمكن حذفها

### عرض الصلاحيات في الجدول

- يتم عرض الصلاحيات المرتبطة بالدور كقائمة مفصولة بفواصل
- إذا لم تكن هناك صلاحيات مرتبطة بالدور، يظهر "لا شيء" (None)
- يمكن تحسين العرض في المستقبل لعرض الصلاحيات كـ Badges أو Tooltips

## التحديثات المستقبلية المقترحة

1. **إضافة حقول العناوين**: إضافة حقول `title_ar` و `title_en` في نموذج إضافة/تعديل الدور
2. **صفحة تعديل**: إضافة صفحة أو Modal لتعديل الأدوار (خاصة الصلاحيات المرتبطة والعناوين)
3. **عرض العناوين**: عرض العناوين في الجدول بدلاً من أو بجانب الأسماء
4. **تصفية متقدمة**: إضافة خيارات تصفية إضافية (حسب المستأجر، حسب الصلاحيات، إلخ)
5. **عرض المستخدمين**: إضافة عمود أو صفحة لعرض المستخدمين المرتبطين بالدور
6. **تصدير/استيراد**: إضافة إمكانية تصدير واستيراد الأدوار
