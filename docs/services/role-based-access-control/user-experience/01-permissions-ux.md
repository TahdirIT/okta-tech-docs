# تجربة المستخدم - إدارة الصلاحيات

## الوصول إلى الصفحة

يتم الوصول إلى صفحة إدارة الصلاحيات من قائمة الإعدادات الرئيسية، أو من خلال الرابط المباشر `/settings/permissions`.

## عرض قائمة الصلاحيات

عند فتح الصفحة، يرى المستخدم:

### القسم الأيسر (إنشاء وإدارة الصلاحيات)

**بطاقة إنشاء صلاحية جديدة:**
- حقل إدخال **اسم الصلاحية** (مطلوب، مثل: `manage-users`)
- قائمة منسدلة **النطاق** (مطلوب: `tenant` أو `system`)
- زر **إضافة صلاحية** لإنشاء الصلاحية الجديدة

**بطاقة مسح واستيراد الصلاحيات:**
- زر **مسح** (أيقونة بحث) لمسح الكود لاكتشاف الصلاحيات المفقودة
- عرض **تاريخ آخر مسح** (إن وجد)
- قائمة **الصلاحيات المفقودة من الكود**:
  - مربعات اختيار لكل صلاحية مفقودة
  - مربع اختيار **تحديد الكل** لتحديد جميع الصلاحيات
- أزرار الإجراءات (تظهر فقط عند وجود صلاحيات مفقودة):
  - **إضافة جميع المفقودة إلى قاعدة البيانات**
  - **إضافة المحددة إلى قاعدة البيانات** (يظهر فقط عند تحديد صلاحيات)

### القسم الأيمن (عرض الصلاحيات)

**جدول الصلاحيات:**
- عمود **الاسم**: اسم الصلاحية
- عمود **النطاق**: نطاق الصلاحية (`TENANT` أو `SYSTEM`) مع شارة ملونة
- عمود **الإجراءات**: زر **حذف** لكل صلاحية

**حالات الجدول:**
- عند وجود صلاحيات: عرض جميع الصلاحيات مرتبة حسب النطاق ثم الاسم
- عند عدم وجود صلاحيات: رسالة "لا توجد صلاحيات بعد"

## إنشاء صلاحية جديدة

### الخطوات:

1. المستخدم يملأ حقل **اسم الصلاحية** (مثل: `manage-users`)
2. المستخدم يختار **النطاق** من القائمة المنسدلة:
   - `tenant`: صلاحية خاصة بالمستأجر
   - `system`: صلاحية على مستوى النظام
3. المستخدم ينقر على زر **إضافة صلاحية**
4. يتم التحقق من البيانات:
   - يجب أن يكون الاسم فريداً ضمن نفس `guard_name` و `scope`
   - يجب أن يكون الاسم مطلوباً ولا يتجاوز 120 حرفاً
5. بعد النجاح:
   - يتم إضافة الصلاحية إلى قاعدة البيانات
   - يتم إعادة تعيين حقل الاسم
   - يظهر إشعار نجاح: "تم حفظ الصلاحية بنجاح"
   - يتم تحديث الجدول تلقائياً لعرض الصلاحية الجديدة

## مسح الكود لاكتشاف الصلاحيات المفقودة

### الخطوات:

1. المستخدم ينقر على زر **مسح** (أيقونة البحث)
2. يتم تنفيذ عملية المسح:
   - مسح جميع ملفات المشروع (Blade, PHP)
   - البحث عن أنماط الصلاحيات المستخدمة في الكود
   - مقارنة الصلاحيات المكتشفة مع الصلاحيات الموجودة في قاعدة البيانات
   - حفظ التقرير في `storage/app/rbac/missing-rbac.json`
3. بعد اكتمال المسح:
   - يتم تحديث قائمة **الصلاحيات المفقودة**
   - يتم تحديث **تاريخ آخر مسح**
   - يتم إعادة تعيين التحديدات السابقة

## استيراد الصلاحيات المفقودة

### الطريقة الأولى: استيراد جميع الصلاحيات

1. المستخدم ينقر على زر **إضافة جميع المفقودة إلى قاعدة البيانات**
2. يتم استيراد جميع الصلاحيات المفقودة إلى قاعدة البيانات
3. بعد النجاح:
   - يظهر إشعار نجاح: "تم استيراد الصلاحيات بنجاح"
   - يتم تحديث الجدول تلقائياً
   - يتم إعادة تعيين قائمة الصلاحيات المفقودة

### الطريقة الثانية: استيراد الصلاحيات المحددة

1. المستخدم يحدد الصلاحيات المطلوبة من القائمة (يمكن استخدام **تحديد الكل**)
2. المستخدم ينقر على زر **إضافة المحددة إلى قاعدة البيانات**
3. يتم استيراد الصلاحيات المحددة فقط
4. بعد النجاح:
   - يظهر إشعار نجاح: "تم استيراد الصلاحيات بنجاح"
   - يتم تحديث الجدول تلقائياً
   - يتم إعادة تعيين التحديدات

## حذف صلاحية

### الخطوات:

1. المستخدم ينقر على زر **حذف** بجانب الصلاحية المطلوبة
2. يتم التحقق من عدم ارتباط الصلاحية بأدوار أو مستخدمين
3. بعد التأكيد:
   - يتم حذف الصلاحية (Soft Delete)
   - يظهر إشعار نجاح
   - يتم تحديث الجدول تلقائياً

### القيود:

- لا يمكن حذف صلاحية مرتبطة بأدوار
- لا يمكن حذف صلاحية مرتبطة بمستخدمين مباشرة

## حالات خاصة

### عدم وجود صلاحيات

عند عدم وجود صلاحيات في النظام:
- يظهر في الجدول رسالة: "لا توجد صلاحيات بعد"
- تظهر قائمة الصلاحيات المفقودة فارغة مع رسالة: "لم يتم العثور على صلاحيات مفقودة. قم بتشغيل المسح للتحديث"

### عدم وجود صلاحيات مفقودة

عند عدم وجود صلاحيات مفقودة:
- تظهر رسالة: "لم يتم العثور على صلاحيات مفقودة. قم بتشغيل المسح للتحديث"
- لا تظهر أزرار الاستيراد
