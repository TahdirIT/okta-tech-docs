# الأمان والحدود

## سياسات أساسية مقترحة

- **TTL**: صلاحية قصيرة (مثال: 5 دقائق) مع إمكانية تخصيصها حسب `purpose`.
- **Max attempts**: عدد محاولات تحقق محدود (مثال: 5 محاولات).
- **Rate limiting**:
  - لكل وجهة (destination) ضمن نافذة زمنية.
  - لكل مستخدم/جلسة (إن وجد).
  - لكل عنوان IP كطبقة حماية إضافية.

## ممارسات أمان

- عدم تخزين الرمز بصيغة صريحة (يفضل تخزين بصمة/Hash).
- إرجاع رسائل خطأ غير كاشفة (لا تفضح إن كانت الوجهة موجودة).
- تسجيل/تدقيق (Audit) لكل:
  - طلب إرسال
  - نجاح/فشل تحقق
  - استهلاك الرمز
