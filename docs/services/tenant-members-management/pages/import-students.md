# صفحة استيراد الطلاب (Import Students)

## الهدف

إضافة/تحديث طلاب الجهة عبر ملف، مع:

- معاينة البيانات قبل الحفظ.
- تحقق من القيود (مثل حد المقاعد، منع الطالب في جهة أخرى).
- إلزام ربط الطالب بالصف/الفصل.
- (اختياري) استيراد أولياء الأمور وربطهم بالطلاب.

## القنوات المدعومة (Conceptual)

- **ملف طلاب**: يحتوي بيانات الطالب + ربطه الأكاديمي.
- **ملف أولياء الأمور**: قد يكون ملفاً مكملاً لإنشاء/ربط ولي الأمر.
- **قوالب مختلفة**: حسب بلد/نظام/مصدر بيانات (يتم التحكم بمخططها عبر إعدادات System Admin).

## تجربة المستخدم

- رفع الملف → Parsing → عرض قائمة صفوف للتحقق.
- تمييز الأخطاء على مستوى الصف (Row-level errors):
  - نقص الهوية
  - نقص الصف أو الفصل
  - صيغة هاتف غير صحيحة
  - طالب مرتبط بجهة أخرى
- زر “حفظ/تنفيذ الاستيراد”:
  - ينفذ Upsert للمستخدمين
  - ينشئ/يحدث سجلات الطلاب
  - ينشئ روابط الطالب↔الجهة و الطالب↔الفصل
  - ينشئ/يربط أولياء الأمور (إن توفر)

## ضوابط إلزامية (Mandatory)

- كل طالب يجب أن يحمل `stage_id` و`section_id` صالحين.
- لا يُسمح بالحفظ إذا وُجد صف واحد على الأقل ينتهك الإلزام.

## نتائج التنفيذ

- رسالة نجاح + إعادة توجيه إلى قائمة الطلاب.
- تشغيل Jobs لاحقة عند الحاجة:
  - ضبط كلمات المرور الافتراضية
  - تنظيف العلاقات المكررة
