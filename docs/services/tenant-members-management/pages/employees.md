# صفحة الموظفين (Employees)

## الهدف

تمكين مسؤول الجهة من:

- عرض قائمة الموظفين داخل الجهة.
- البحث بالاسم/الهوية/الهاتف.
- إضافة موظف (يدوي أو استيراد).
- تعديل بيانات موظف.
- تعيين الأدوار والصلاحيات المرتبطة به.
- تغيير كلمة المرور (سياسة المنصة).
- فصل الموظف من الجهة (Release membership).

## عناصر الصفحة

- **بحث** + فلاتر اختيارية (مثل الدور أو الحالة).
- **جدول**:
  - الاسم + الهوية
  - الوظيفة/النوع (Teacher/Administrator…)
  - الدور الأساسي + أدوار إضافية
  - إجراءات: تعديل / تغيير كلمة المرور / فصل

## سلوك إضافة موظف

- إدخال الهوية الوطنية/الإقامة للتحقق من:
  - هل يوجد حساب مستخدم مسبقاً؟
  - هل هو مرتبط بالفعل بهذه الجهة؟
  - هل هو مرتبط بجهة أخرى وفق سياسة النظام؟
- عند الإضافة:
  - إنشاء/ربط employee membership داخل الجهة
  - تعيين دور أساسي + (اختياري) أدوار إضافية

## فصل الموظف

- فصل العضوية لا يحذف الموظف ولا المستخدم.
- يسجل `released_at` على علاقة employee↔tenant.
- (اختياري حسب سياسة RBAC) تنظيف الأدوار tenant-scoped.
