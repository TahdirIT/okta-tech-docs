# صفحة تهيئة مخططات ملفات الاستيراد (System Admin — File Formats)

## الهدف

تمكين مسؤول النظام من تعريف/تعديل **مخططات قراءة الملفات** التي ترفعها الجهات، بحيث يصبح الاستيراد:

- قابلاً للتكيف حسب البلد/السياق.
- مستقراً عند اختلاف مواضع الأعمدة/الأسطر في ملفات Excel.
- قادراً على تنزيل قوالب فارغة (Empty templates) لتوحيد الإدخال.

## نطاق الصفحة

- إدارة مخططات لعدة أنواع ملفات، مثل:
  - Students file
  - Students data file
  - Guardians file
  - Employees file
  - Okta template (download empty + upload filled)

## عناصر المخطط (Schema) — مفاهيمياً

- `start_row`: رقم السطر الذي يبدأ منه المحتوى.
- `increments`: مقدار الانتقال بين “السجلات” إذا كانت البيانات موزعة على صفوف متكررة.
- تعريف كل حقل كـ tuple:
  - `x`: رقم العمود
  - `y_offset`: إزاحة صفية
  - `relative`: هل الإزاحة نسبية للصف الحالي أم مطلقة؟

مثال تخطيطي:

```json
{
  "start_row": 3,
  "increments": 1,
  "fields": {
    "student_name": { "x": 2, "y_offset": 0, "relative": true },
    "national_id": { "x": 5, "y_offset": 0, "relative": true }
  }
}
```

## تنزيل قالب Okta (Empty Excel)

القالب الفارغ يجب أن يحتوي:

- أعمدة قياسية للطالب/ولي الأمر.
- أعمدة ربط أكاديمي:
  - `stage_id` أو `stage_name`
  - `section_id` أو `section_name`

> مبدأ: القالب يساعد مسؤول الجهة على تعبئة بيانات سليمة لتحقيق إلزامية المرحلة/الفصل.

## رفع ملف Okta (Filled Excel)

عند رفع الملف:

- اختيار المخطط المناسب (Schema) تلقائياً حسب البلد/السياق.
- Parsing ثم Validation ثم preview.
- لا يتم التنفيذ النهائي إلا بعد نجاح قواعد الإلزام.
