# صفحة الطلاب (Students)

## الهدف

تمكين مسؤول الجهة من:

- عرض قائمة الطلاب داخل الجهة.
- البحث والتصفية (حسب حالة الطالب/الصف/الفصل).
- إضافة طالب (يدوياً أو عبر استيراد).
- فتح تفاصيل الطالب.
- تعديل بيانات الطالب الأساسية.
- ربط ولي أمر بالطالب أو إنشاء ولي جديد.
- فصل الطالب من الجهة.

## عناصر الصفحة

- **شريط أدوات**:
  - بحث نصي: الاسم/الهوية.
  - زر “إضافة طلاب” بقنوات:
    - إضافة يدوي
    - استيراد عبر ملف

- **فلاتر**:
  - حالة الطالب (مثال: regular / affiliate)
  - الصف/المرحلة (Stage)
  - الفصل (Section) — يظهر بعد اختيار الصف

- **جدول الطلاب**:
  - الاسم + معرف الهوية
  - الصف
  - الفصل
  - الحالة
  - (اختياري) عداد أدوات/خدمات مرتبطة
  - ولي الأمر (إن وجد) + زر ربط ولي عند عدم وجوده
  - قائمة إجراءات

## السلوك (Behavior)

- **Search**: تحديث القائمة بتأخير (debounce) وتصفير pagination.
- **Filter chaining**:
  - تغيير الصف يعيد ضبط اختيار الفصل.
- **إجراءات على الصف**:
  - فتح التفاصيل.
  - تعديل بيانات الطالب (Modal).
  - ربط/عرض ولي الأمر (Modal).
  - فصل الطالب (Modal تأكيد).

## أخطاء متوقعة

- محاولة إضافة طالب موجود بالفعل داخل الجهة.
- محاولة إضافة طالب مرتبط بجهة أخرى (حسب سياسة النظام).
- الوصول إلى حد المقاعد (Seat limit) — يجب عرض رسالة واضحة وتوجيه للترقية.
