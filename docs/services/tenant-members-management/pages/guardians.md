# صفحة أولياء الأمور (Guardians)

## الهدف

تمكين مسؤول الجهة من:

- عرض قائمة أولياء الأمور المرتبطين بطلاب داخل الجهة.
- البحث بالاسم/الهوية/الهاتف.
- عرض ولي الأمر وتحديث بياناته.
- إدارة الأبناء المرتبطين (إضافة ربط/فصل ربط).

## عناصر الصفحة

- **بحث**: (name / national_id / phone)
- **جدول**:
  - اسم ولي الأمر
  - الهوية + الهاتف
  - عدد الأبناء المرتبطين
  - عرض أبنائه (Preview) + رابط “عرض التفاصيل/تعديل”

## سلوك إدارة ولي الأمر (Modal / Details)

- **عرض بيانات ولي الأمر** (قابلة للتعديل إن كان مسموحاً)
- **قائمة الأبناء المرتبطين**:
  - إظهار الأبناء ضمن الجهة الحالية (Tenant)
  - فصل الارتباط (يسجل `unlinked_at`)
- **ربط ابن جديد**:
  - بحث عن الطالب داخل الجهة (بالاسم/الهوية)
  - اختيار الطالب ثم إنشاء العلاقة guardian↔student

## قيود وأخطاء

- منع ربط ولي الأمر بالطالب إذا كان الحساب نفسه (self-link).
- منع ازدواجية الربط إذا كانت العلاقة فعّالة مسبقاً.
- عدم إظهار طلاب خارج نطاق الجهة.
