# إدارة أعضاء الجهة (Tenant Members Management)

توثّق هذه الخدمة إدارة **أعضاء الجهة التعليمية (Tenant)** كبيانات تشغيلية مشتركة داخل نطاق الجهة، وتشمل:

- **الطلاب**: إنشاء/ربط/نقل/فصل الطالب، مع إلزام ربطه بصف/فصل (Grade/Section).
- **أولياء الأمور**: إنشاء الحساب وربطه بأبنائه، وإدارة حالة الارتباط.
- **الموظفون**: إضافة موظف وربطه بالجهة، وتعيين دور أساسي وأدوار إضافية (مثل “معلم” و/أو “إداري”).
- **الصفوف والفصول**: إدارة البنية التعليمية التي يعتمد عليها ربط الطالب (Stages/Sections).
- **الاستيراد عبر ملفات**: خطوط استيراد الطلاب/أولياء الأمور/الموظفين عبر ملفات (مثل Excel)، مع دعم مخططات قابلة للتهيئة حسب السياق.
- **تهيئة مخططات الملفات (System-level)**: إدارة “Schema” لقراءة الملفات (تنزيل قالب فارغ، وتحديد مواضع الأعمدة/الأسطر)، مع تطبيقها لاحقاً ضمن عمليات الجهة.

> ملاحظة تصميمية: هذا التوثيق يصف **الخدمة في معمارية النظام المستهدف** فقط، ويعرض “اعتماديات” بشكل مفاهيمي دون شرح مصادر تنفيذ.

## نطاق الخدمة (Boundaries)

- **داخل النطاق**:
  - إدارة الهوية التشغيلية للعضو داخل الجهة (Student/Guardian/Employee).
  - ربط العضويات بعلاقات Tenant-scoped (مثل الطالب↔فصل، الطالب↔ولي، الموظف↔جهة).
  - تعريف Use-cases واضحة للواجهة (Pages) والاستيراد.
  - قواعد إلزامية الربط الأكاديمي للطلاب.
  - قواعد صلاحيات الوصول على مستوى الجهة.

- **خارج النطاق** (اعتماديات مفاهيمية فقط):
  - إدارة المستخدم العام وبياناته الأساسية (User Management).
  - RBAC وتخزين الصلاحيات/الأدوار كمنظومة (Role-Based Access Control).
  - الاشتراكات/قيود المقاعد (Subscription/Seat Limits).
  - القنوات (إشعارات/رسائل/OTP) إلا بقدر ما تتأثر بها إجراءات إنشاء الحساب أو تنشيطه.

## مبادئ تصميم (Design Principles)

- **تعددية المستأجرين (Tenant Isolation)**: كل عمليات الربط والفصل والبحث تتم ضمن سياق الجهة الحالي.
- **إلزام البنية الأكاديمية للطالب**: لا يُسجَّل الطالب “نشطاً” داخل الجهة بدون صف/فصل صالحين.
- **فصل “User” عن “Member”**: المستخدم العام قد يكون له أكثر من تمثيل تشغيلي (طالب/ولي/موظف) حسب السياق، مع قواعد واضحة لمنع التعارض.
- **سجل علاقات قابل للتدقيق**: عمليات الربط/الفصل تؤثر على تاريخ العلاقة (Release/Unlink) بدل حذف مباشر.
- **استيراد قابل للتهيئة**: قراءة الملفات تعتمد على “Schema” قابلة للتعديل على مستوى النظام/الدولة، ولا تُدمج بشكل ثابت داخل منطق الاستيراد.

## مكونات عليا

- **Pages (Tenant Admin)**:
  - قائمة الطلاب + إضافة/تعديل/تفاصيل + ربط ولي + فصل طالب.
  - قائمة أولياء الأمور + إضافة/تعديل + إدارة الأبناء المرتبطين.
  - قائمة الموظفين + إضافة/تعديل + تغيير كلمة المرور + فصل موظف.
  - أدوار الموظفين داخل الجهة (Roles/Permissions).
  - استيراد الطلاب (Noor/Tahdir أو قوالب) + استيراد الموظفين.

- **System Admin Configuration**:
  - صفحة/واجهة تهيئة مخططات ملفات الاستيراد (Per country / per context).
  - تنزيل قالب Excel فارغ “مُرمّز” لتسهيل اختيار الصف/الفصل.

## مصطلحات

- **Tenant**: جهة/مدرسة/كيان يملك عزلاً للبيانات والصلاحيات.
- **Stage/Grade**: الصف/المرحلة.
- **Section/Class**: الفصل داخل الصف.
- **Link / Unlink**: ربط/فصل علاقة تشغيلية بين كيانين (مع حفظ التاريخ).
