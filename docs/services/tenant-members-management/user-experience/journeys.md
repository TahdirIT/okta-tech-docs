# الرحلات (Journeys) — Tenant Members Management

## رحلة 1: مسؤول الجهة يضيف طالباً جديداً

- **الدافع**: إدخال طالب جديد للجهة قبل بدء العام الدراسي.
- **الخطوات**:
  - فتح صفحة الطلاب.
  - إضافة يدوي → إدخال رقم الهوية.
  - النظام يتحقق من:
    - وجود الطالب داخل الجهة
    - أو ارتباطه بجهة أخرى
  - إدخال الاسم والبيانات الأساسية (إن لزم).
  - اختيار الصف ثم الفصل (إلزامي).
  - حفظ → يظهر الطالب في القائمة مع فصل فعّال.

- **نقاط ألم محتملة**:
  - عدم وجود الصف/الفصل في النظام → يحتاج مسؤول الجهة لإضافتها من صفحة الصفوف/الفصول.

## رحلة 2: مسؤول الجهة يستورد الطلاب من ملف

- **الدافع**: إدخال دفعة كبيرة بسرعة.
- **الخطوات**:
  - فتح صفحة الاستيراد.
  - رفع الملف.
  - النظام يقوم بـ parsing ويعرض preview مع أخطاء الصفوف.
  - المسؤول يصحح الأخطاء أو يحذف الصفوف غير الصحيحة.
  - تنفيذ الاستيراد.

- **نقاط إلزام**:
  - أي صف لا يحتوي صف/فصل صالحين يمنع التنفيذ النهائي.

## رحلة 3: إدارة ولي الأمر وربط طالب جديد

- **الدافع**: إضافة ابن جديد لولي أمر موجود.
- **الخطوات**:
  - فتح صفحة أولياء الأمور.
  - البحث عن ولي الأمر.
  - فتح تفاصيله.
  - البحث عن الطالب داخل الجهة.
  - ربط الطالب → تظهر العلاقة فوراً.

## رحلة 4: إدارة موظف وتوسيع صلاحياته

- **الدافع**: موظف أصبح لديه مهام إدارية إلى جانب التدريس.
- **الخطوات**:
  - فتح صفحة الموظفين.
  - فتح الموظف → تعديل الأدوار.
  - تعيين “role” إضافي أو تعديل الدور الأساسي.
  - حفظ → تتغير الصلاحيات ضمن نفس الجهة.
