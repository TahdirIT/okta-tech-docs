# معالجة البيانات (Data Handling) — Overview

## نطاق هذه الحزمة

تصف هذه الحزمة كيفية تعامل الخدمة مع:

- إدخال بيانات الأعضاء (طلاب/أولياء/موظفين) عبر UI وAPI.
- الاستيراد عبر ملفات (Excel/CSV) مع مخطط قابل للتهيئة.
- upsert وربط العلاقات التاريخية عبر pivots.
- jobs لاحقة للتنظيف/التهيئة.

## مبادئ

- **Schema-driven parsing**: لا يعتمد parsing على “أعمدة ثابتة”، بل على schema قابل للتعديل.
- **Preview-first**: قبل التنفيذ النهائي، تُعرض بيانات الاستيراد مع أخطاء الصفوف.
- **Atomic writes**: عمليات الربط/النقل الحساسة تتم داخل transaction.
- **Idempotency**: الاستيراد يعاد تشغيله بأمان عبر upsert وتجنب التكرار.
