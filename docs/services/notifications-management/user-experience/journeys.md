# رحلات المستخدم (User Journeys)

## مسؤول المنصة (Super Admin)
- يضيف تعريفات إشعارات أساسية للمنصة.
- يعرّف قوالب افتراضية متعددة اللغات ويضبط قواعد النشر.
- يحدد اللغة الافتراضية للإشعارات حسب الدولة.

## مسؤول الجهة (Tenant Admin)
- يختار التعريفات المتاحة ويخصص ما يحتاجه.
- يغير اللغة الافتراضية للإشعارات على مستوى الجهة إن لزم.
- يحرر قوالب مخصّصة تلائم هوية الجهة مع المعاينة.

## المستخدم النهائي
- يتلقى الإشعارات عبر القنوات المختارة (بريد/رسائل/دَفع/داخل النظام).
- يدير التفضيلات (اشتراك/إلغاء اشتراك) حسب التعريف والقناة.
- يطالع مركز الإشعارات ويضبط الحالات (مقروء/مثبّت/مؤرشف).
