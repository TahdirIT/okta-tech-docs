# الحالات الحديّة (Edge Cases)

- فشل قناة معيّنة: يتم التحويل التلقائي لقناة بديلة إن كانت المعرفة تسمح، مع تسجيل السبب.
- نقص بيانات القالب: يتم التحقق المسبق من المتطلبات وإرجاع خطأ قابل للتشخيص قبل الإرسال.
- لغة غير مدعومة للمستخدم: تُطبّق سياسة اللغة الهرمية لاختيار بديل مناسب.
- تجاوزات الجهة المخالفة للمنصة: لا يُسمح بإلغاء حقول أو سياسات إلزامية محددة على مستوى المنصة.
- معدلات إرسال مرتفعة: تطبيق قيود وتجزئة الإرسال (Batching) وإعادة المحاولة التدريجية.
- ازدواجية إرسال: مفتاح خصائص (Idempotency Key) لكل عملية إرسال يمنع التكرار العرضي.
- إلغاء الاشتراك: احترام تفضيلات المستخدم لكل تعريف/قناة مع سجلات تدقيقية.
