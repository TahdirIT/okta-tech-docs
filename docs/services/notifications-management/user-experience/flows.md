# تدفقات الاستخدام (Flows)

## 1) إعداد تعريف إشعار جديد (منصة)
1. مسؤول المنصة يضيف تعريف إشعار جديد.
2. يحدد: مفتاح التعريف، السياق/الحدث، نوع الجمهور/المتلقّي، والقنوات المدعومة.
3. يربط التعريف بقوالب افتراضية متعددة اللغات لكل قناة.
4. ينشر التعريف ليصبح متاحاً للجهات.

## 2) تخصيص اللغة الافتراضية (دولة)
1. مسؤول المنصة يحدد اللغة الافتراضية للإشعارات لكل دولة.
2. تعتمد الجهات داخل الدولة هذه اللغة ما لم تُجاوز على مستوى الجهة.

## 3) تخصيص القوالب/اللغة (الجهة)
1. مسؤول الجهة يختار التعريفات المتاحة.
2. يخصص اللغة الافتراضية للجهة إن لزم.
3. ينشئ/يعدل قوالباً مخصّصة للقنوات المطلوبة (مع المعاينة).

## 4) إرسال إشعار مبرمج عبر حدث
1. حدث (Domain Event) يطلق إرسال إشعار مرتبط بتعريف.
2. تُسترجع سياسة اللغة للمستلم.
3. يُختار القالب المناسب (لغة/قناة) ويُطبّق عليه البيانات.
4. تُسجّل عملية الإرسال وتُدار إعادة المحاولة عند الفشل.

## 5) عرض الإشعارات داخل النظام
1. المستخدم يفتح مركز الإشعارات.
2. تُجلب الإشعارات غير المقروءة/الحديثة مع التصفية.
3. يمكن وضع علامة مقروء/تثبيت/أرشفة/حذف وفق الصلاحيات.
