# المعمارية التقنية (Architecture)

## نظرة عامة
- طبقة التعريفات (Definitions): تعريف نوع الإشعار، سياق الحدث، القنوات المسموحة، متطلبات البيانات.
- طبقة القوالب (Templates): تمثيل متعدد اللغات والوسائط لكل قناة، مع متغيرات ديناميكية.
- طبقة سياسات اللغة (Locale Policies): تحديد اللغة الفعلية لكل إرسال وفق التسلسل الهرمي (منصة ← دولة ← جهة ← مستخدم).
- طبقة الإرسال (Dispatching): اختيار القناة، تطبيق القالب، الجدولة، الطوابير، إعادة المحاولة.
- طبقة التفضيلات (Preferences): إدارة اشتراك/إلغاء الاشتراك لكل تعريف/قناة.
- طبقة السجلات والمراقبة (Logging & Observability): تتبع الحالة والمقاييس والتنبيهات على الفشل.

## حدود الخدمة (Boundaries)
- مدخلات: أحداث نطاقية (Domain Events)، استدعاءات API صريحة (Trigger APIs).
- مخرجات: موصلات قنوات (Email via SMTP / SMS via taqnyat-sms-laravel / App Notifications via Firebase Messaging)، سجلات، مقاييس، تنبيهات فشل.
- عقود واجهات واضحة لكل قناة قابلة للاستبدال والتوسعة.

## مبادئ التكيّف
- إعدادات مركزية قابلة للتخصيص هرميّاً.
- عدم ربط القناة أو المزوّد داخل المنطق الأساسي؛ يُحقن عبر واجهات.
- نمط Use-Case صغير لكل وظيفة (وفق معيار Feature Services Structure).
- Idempotency لكل عملية إرسال لتجنب التكرار.
