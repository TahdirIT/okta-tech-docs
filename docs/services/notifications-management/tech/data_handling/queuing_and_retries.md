# التعامل مع البيانات: الطوابير وإعادة المحاولة

## الطوابير
- كل إرسال يتم عبر طابور مخصص لكل قناة أو طابور عام مع مفاضلة.
- دعم تجزئة الإرسال (Batching) للقنوات التي تستفيد منه.

## إعادة المحاولة
- استراتيجية خطية/أسية مع حد أقصى للمحاولات.
- تمييز الأخطاء القابلة لإعادة المحاولة مقابل الدائمة.
- تسجيل تفصيلي لكل محاولة في `DeliveryLog`.

## Idempotency
- مفتاح تماثل فريد (تعريف + مستلم + سياق) يمنع الازدواج.
