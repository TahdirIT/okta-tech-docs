# Data handling - Tenant Registration

## مصادر البيانات

خدمة تسجيل الكيان تعتمد على:

- بيانات الدولة + إعداداتها (Country settings)
- إعدادات “تخصيصات تسجيل الكيان” على مستوى الدولة:
  - `docs/services/contries-management/guide/entity-registration-customizations.md`
- إعدادات “المراحل” على مستوى الدولة (عند الحاجة لاختيار مرحلة):
  - `docs/services/contries-management/guide/stages-management.md`

## مدخلات التسجيل

المدخلات الرئيسية المتوقعة:

- `country_id`
- `tenant_type`
- `education_level_group_id` (مطلوب فقط لأنواع: **مدرسة إدارية** و **معلم فردي**)
- `tenant_name`
- `custom_fields` (key/value بحسب تعريف الدولة والنوع)
- `owner`:
  - `username`
  - `name`
  - `national_id`
  - `mobile` (قد يكون مطلوبًا أو اختياريًا)
  - `email` (قد يكون مطلوبًا أو اختياريًا)
  - `password`
  - `password_confirmation`

## التحقق (Validation) على مستوى الخدمة

- **تحقق الدولة**: `country_id` صالح ومتاح للتسجيل.
- **تحقق نوع الكيان**: النوع مفعّل في الدولة.
- **تحقق المرحلة** (لبعض الأنواع فقط):
  - عند `tenant_type` = مدرسة إدارية أو معلم فردي: `education_level_group_id` إلزامي.
  - يجب أن تكون المجموعة ضمن مجاميع مراحل الدولة المحددة (حسب هيكلة الدولة).
- **تحقق الحقول المخصصة**:
  - كل حقل موجود في التعريف.
  - الأنواع مطابقة (number/date/select/…).
  - الخيارات ضمن القيم المسموح بها (للقوائم).
  - `required_when_visible` مطبق بعد تقييم شروط الظهور.
- **تحقق بيانات المستخدم**:
  - تطبيق إلزامية اسم المستخدم/الهوية/الجوال/البريد حسب إعدادات الدولة.
  - تحقق صيغة الجوال/البريد.
  - تحقق رقم الهوية الوطنية (إن وجدت قواعد لكل دولة).
  - منع التكرار: التأكد من عدم تسجيل (رقم الهوية/اسم المستخدم/الجوال/الإيميل) مسبقًا عند إنشاء حساب جديد.
- **تحقق كلمة المرور**: السياسة + التطابق مع التأكيد.

## ناتج التسجيل (Output)

بعد النجاح يتم إنشاء:

- Tenant
- المستخدم المالك
- تخزين `custom_fields` في مكان مناسب (JSON أو جدول منفصل) بحسب قرار التصميم.

ثم:

- ضبط حالة التفعيل `pending_verification`
- إطلاق الأحداث المعرفة في:
  - `docs/services/tenant-registration/guide/05-events.md`
