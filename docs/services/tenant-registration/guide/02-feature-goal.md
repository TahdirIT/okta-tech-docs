# الهدف من الميزة - تسجيل الكيان

## الهدف

توفير تجربة تسجيل واضحة وآمنة لإنشاء Tenant جديد مع:

- تقليل الاحتكاك للمستخدم النهائي (Wizard خطوة بخطوة).
- مرونة عالية حسب الدولة عبر الحقول المخصصة (Custom fields).
- حماية الحسابات عبر **اشتراط التحقق** من الجوال أو البريد قبل السماح بأي عمليات داخل النظام.
- قابلية توسع مستقبلية عبر **أحداث Event-Driven** يمكن لخدمات أخرى الاستماع لها.

## نطاق الميزة (Scope)

يشمل:

- اختيار الدولة (Country).
- اختيار نوع الكيان (Tenant type) من الأنواع المفعلة في الدولة.
- إدخال اسم الكيان (Tenant name).
- إدخال الحقول المخصصة التي تفرضها/تعرضها الدولة حسب نوع الكيان.
- إدخال بيانات المستخدم (المالك): اسم المستخدم، الاسم، رقم الهوية الوطنية، الجوال، البريد.
- كلمة المرور + تأكيد كلمة المرور.
- إنشاء الحساب والكيان مع وضع “غير مفعّل” إلى أن يكتمل التحقق (Mobile/Email verification).

## خارج النطاق (Out of scope)

- آليات التحقق التفصيلية (OTP providers، قنوات إرسال…): تُوثق في خدمة التحقق/الهوية عند توفرها.
- عمليات إدارة الكيان بعد التسجيل (إعدادات، مستخدمون، صلاحيات…): تخص خدمات أخرى.
