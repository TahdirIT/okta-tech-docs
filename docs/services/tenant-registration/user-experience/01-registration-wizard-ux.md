# تجربة المستخدم - تدفق تسجيل كيان جديد (Wizard)

هذا المستند يصف واجهة وتدفق المستخدم النهائي لإنشاء Tenant جديد.

المرجع الخاص بتخصيصات الدولة:

- `docs/services/contries-management/guide/entity-registration-customizations.md`

## الهدف

إتاحة تسجيل كيان جديد عبر خطوات متتابعة تقلل الأخطاء وتسمح بعرض حقول ديناميكية حسب الدولة ونوع الكيان.

## مكونات عامة في الواجهة

- مؤشر خطوات (Stepper).
- زر “التالي” و “السابق”.
- حفظ مؤقت لمدخلات المستخدم داخل الجلسة (Session/Local state) حتى نهاية التسجيل.
- رسائل أخطاء واضحة لكل حقل.

## الخطوة 1: اختيار الدولة

- **العنصر**: قائمة الدول (Select).
- **سلوك**:
  - عند الاختيار: تحميل إعدادات التسجيل الخاصة بالدولة:
    - إلزامية الهوية الوطنية/الجوال/البريد.
    - أنواع الكيانات المفعلة.
    - الحقول المخصصة وتعريفها.

## الخطوة 2: اختيار نوع الكيان

- **العنصر**: قائمة/بطاقات لأنواع الكيانات (Cards أو Select).
- **سلوك**:
  - عرض الأنواع المفعلة فقط.
  - الترتيب ثابت (مرجع `docs/tenants.md`).

## الخطوة 3: اختيار المرحلة (Stage) — لبعض الأنواع فقط

### متى تظهر هذه الخطوة؟

تظهر فقط عندما يختار المستخدم نوع كيان:

- **مدرسة إدارية**
- **معلم فردي**

### مصدر بيانات المراحل

قائمة المراحل تُستمد من إعدادات الدولة وفق المفهوم الموضح في:

- `docs/services/contries-management/guide/stages-management.md`

### تجربة الاختيار

- **العنصر**: قائمة منسدلة (Select) أو Tree picker (اختياري) يعرض:
  - اسم المرحلة (رسمي/مفهوم للمستخدم)
  - الاسم المختصر إن وجد
  - (اختياري) المجموعة (طفولة مبكرة/ابتدائي/…)
- **تحقق**:
  - مطلوب لهذه الأنواع.

> ملاحظة: إن كانت الدولة لا تملك مراحل معرفة بعد، يجب عرض رسالة واضحة للمستخدم (أو تعطيل متابعة التسجيل) حسب سياسة المنتج.

## الخطوة 4: اسم الكيان

- **العنصر**: حقل نصي لاسم الكيان.
- **تحقق**:
  - مطلوب.
  - طول مناسب.

## الخطوة 5: الحقول المخصصة (Dynamic custom fields)

- **العنصر**: مجموعة حقول تتغير حسب الدولة + نوع الكيان.
- **سلوك**:
  - تطبيق شروط الظهور: إخفاء/إظهار حقول حسب قيم سابقة.
  - عند تغيير قيمة تؤثر على حقول لاحقة: إعادة تقييم الإظهار/الإخفاء ومسح القيم المخفية (حسب السياسة).
- **تحقق**:
  - `required_when_visible` يطبق فقط عندما يكون الحقل ظاهرًا.

## الخطوة 6: بيانات المستخدم (المالك)

### الحقول

- اسم المستخدم
- الاسم
- رقم الهوية الوطنية
- الجوال
- البريد الإلكتروني

### إلزامية الهوية/الجوال/البريد

تعتمد على إعدادات الدولة:

- إن كانت الدولة تفرض الهوية الوطنية: يظهر مؤشر “إلزامي” + يمنع التالي بدون إدخالها.
- إن كانت الدولة تفرض الجوال: يظهر مؤشر “إلزامي” + يمنع التالي بدون إدخاله.
- إن كانت الدولة تفرض البريد: نفس الشيء للبريد.

## الخطوة 7: كلمة المرور

- كلمة المرور
- تأكيد كلمة المرور
- إظهار متطلبات كلمة المرور (Password policy) بشكل مساعد.

## الخطوة 8: مراجعة وإرسال (اختياري)

يمكن إضافة خطوة مراجعة تلخص:

- الدولة
- نوع الكيان
- اسم الكيان
- ملخص الحقول المخصصة
- بيانات المستخدم (بدون كلمة المرور)

ثم زر “إنشاء الحساب/الكيان”.

## النتيجة بعد الإرسال

- إظهار شاشة نجاح “تم إنشاء الحساب”.
- تحويل المستخدم مباشرة إلى شاشة “التحقق” مع رسالة:
  - “لا يمكنك استخدام النظام قبل التحقق من الجوال/البريد”.
