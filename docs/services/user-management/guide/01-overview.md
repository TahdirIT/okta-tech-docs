# نظرة عامة - إدارة المستخدمين

خدمة **إدارة المستخدمين (User Management)** مسؤولة عن إدارة حسابات المستخدمين في المنصة، بما في ذلك:

- إنشاء/تحديث بيانات المستخدم الأساسية (Profile).
- التحقق من قنوات التواصل (جوال/بريد) وعلاقتها بتفعيل الحساب.
- منع إنشاء حسابات مكررة عبر قواعد Uniqueness.

## التكامل مع خدمة تسجيل الكيان (Tenant Registration)

خدمة `tenant-registration` تستخدم `user-management` لإنشاء أو ربط **المستخدم المالك (Owner/Admin)** أثناء تدفق تسجيل كيان جديد:

- **إنشاء حساب جديد**: عندما يختار المستخدم “ليس لدي حساب”.
- **ربط حساب موجود**: عندما يختار المستخدم “لدي حساب” ويتم تسجيل الدخول بنجاح.
- **إكمال بيانات ناقصة**: إذا كان لدى المستخدم حساب موجود لكن تنقصه بيانات إلزامية حسب إعدادات الدولة.

> ملاحظة: قواعد إلزامية الهوية/الجوال/البريد أثناء التسجيل تعتمد على إعدادات الدولة (Country) في:
> - `docs/services/contries-management/guide/entity-registration-customizations.md`

## التكامل مع خدمة التحكم في الوصول (RBAC)

إدارة الأدوار والصلاحيات ليست ضمن `user-management`، لكنها تتكامل معها عبر:

- `docs/services/role-based-access-control/tech/permissions.md` (تعيين الأدوار للمستخدمين)
